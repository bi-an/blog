## 场景

栈回溯里常见这种形式：

```text
/path/to/libxxx.so(+0x8fa860)[0x15182684b860]
```

含义：

- **`libxxx.so`** ：出问题的共享库。
- **`+0x8fa860`** ：相对该库**映射基址**的偏移（ relocation 后的代码地址 − 库基址）。
- **`[0x15182684b860]`** ：当时进程里的**绝对虚拟地址**（随 ASLR 变化，单独用时要配合
  `/proc/<pid>/maps` 里的基址）。

调试时通常先用 **`+0x...` 偏移** 对 **未加载时的 `.so` 文件** 做符号解析。

## 首选：`addr2line`

在带有 **debug info**（`-g`，且未 strip 掉 `.debug_*` 或未分离错 debug 文件）的 `.so` 上：

```bash
addr2line -e /绝对路径/libxxx.so -f -C 0x8fa860
```

| 选项      | 作用                     |
| --------- | ------------------------ |
| `-e FILE` | 指定 ELF（这里是 `.so`） |
| `-f`      | 同时打印函数名           |
| `-C`      | Demangle C++ 符号        |

输出示例：函数名 + 源文件:行号。

若只有绝对地址、且已知该库在进程中的加载基址 `BASE`（例如从 maps 读出）：

```bash
addr2line -e /path/to/libxxx.so -f -C $((0x15182684b860 - BASE))
```

（`BASE` 必须是该映射段的**起始地址**，与栈里 `+offset` 的约定一致。）

## 补充：`nm` / `objdump`（看落在哪个符号里）

无行号信息或想确认符号边界时：

```bash
nm -n --defined-only /path/to/libxxx.so | less
# 或
objdump -t /path/to/libxxx.so | less
```

在按地址排序的列表里，找 **不大于目标偏移的最后一个符号**，即大致「落在哪个函数里」。精确行号仍以
`addr2line`（或 debug 信息）为准。

## C++ 符号 demangle

若某处只给出 mangled 名：

```bash
echo '_ZN12_GLOBAL__N_111userCmdBodyEiPPKclS1_' | c++filt
```

## 常见问题

1. **`??` 或明显不对**

   - `.so` 被 strip、或 debug 在单独的 `.debug` 文件里且工具找不到 → 需要带 debug 的构建产物或配置
     `DEBUGINFOD` 等。

2. **行号与当前源码不一致**

   - 栈对应的二进制与本地源码版本不同 → 以**编该 `.so` 的源码/提交**为准。

3. **PIE / 地址空间**
   - 对「文件里的 `.so`」使用 **`+offset`** 一般即可；用绝对地址时要先减**该库本次运行的加载基址**。

## 最小复现命令模板

```bash
SO=/绝对路径/libxxx.so
OFF=0x8fa860   # 来自 (+0x...) 的十六进制偏移

addr2line -e "$SO" -f -C "$OFF"
```
