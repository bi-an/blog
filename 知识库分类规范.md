## 🧭 一、分类的核心原则

这三条是整个知识库的“宪法”：

1. categories 表示“主题归属”（大类、稳定）

- 数量少（5–10 个）
- 未来不会频繁变化
- 代表“知识的领域”

2. tags 表示“概念标签”（关键词、灵活）

- 数量可以很多
- 一个笔记可以有多个 tag
- 用于搜索、交叉引用

3. 归类看“目的”，标签看“内容”

- 目的：这篇笔记主要解决什么问题？→ category
- 内容：涉及哪些技术名词？→ tags

## 🗂 二、顶层 categories

下面这 7 个大类覆盖你提到的所有内容，并且未来可扩展。

```
debugging/        # 调试方法、问题定位、堆栈、符号表、gdb
performance/      # 性能分析、profiling、优化策略
concurrency/      # 并发、多线程、atomic、锁
cpp/              # C++ 语言技巧、标准库、模板、RAII
system/           # Linux 系统编程、进程、线程、内存模型
tools/            # 工具使用方法（addr2line、objdump、perf）
memory/           # 内存管理、泄漏、分配器、内存模型
misc/             # 暂时无法分类的内容
```

每个目录的职责边界如下。

## 🧩 三、各 category 的职责定义

1. debugging/

用于：解决问题的方法

放：

- 如何定位崩溃
- 如何分析 core dump
- 如何打印堆栈
- 如何排查死锁
- 如何分析符号表
- 调试策略、方法论

判断标准：

如果文章的核心是“解决某类问题”，放 debugging。

2. tools/

用于：工具使用说明

放：

- addr2line 用法
- objdump 用法
- readelf 用法
- perf 命令详解
- gdb 常用命令
- clang-format、clang-tidy

判断标准：

如果文章的核心是“如何使用某个工具”，放 tools。

3. performance/

用于：性能分析与优化

放：

- perf 分析 CPU
- flamegraph
- cache miss 分析
- lock contention
- profiling 方法论

4. concurrency/

用于：并发编程

放：

- mutex、spinlock
- atomic、memory order
- lock-free
- 线程池
- 并发 bug（data race、死锁）

5. cpp/

用于：C++ 语言本身

放：

- RAII
- 智能指针
- 模板技巧
- C++20/23 特性
- ABI、name mangling

6. system/

用于：系统编程知识

放：

- Linux syscall
- 进程 / 线程模型
- 内存布局
- 文件系统
- 网络编程

7. memory/

用于：内存相关的深度内容

放：

- 内存泄漏分析
- 分配器（tcmalloc、jemalloc）
- 内存模型
- 堆、栈、虚拟内存

8. misc/

用于：暂时无法分类的内容

放：

- 临时记录
- 还没整理的内容
- 不确定归类的东西

## 🏷 四、tags 使用规范

tags 是“关键词”，不是分类。

tags 的作用：

- 让你能快速搜索
- 让文章能跨 category 关联
- 不需要提前设计，随写随加

tags 示例：

```
addr2line
backtrace
gdb
perf
flamegraph
atomic
mutex
threadpool
memory-leak
valgrind
jemalloc
tcmalloc
linux
elf
symbol
stacktrace
debug
profiling
optimization
c++20
template
raii
```

tags 的规则：

- 全小写
- 用 - 连接（如 memory-leak）
- 不要用中文（搜索不方便）
- 不要太长（3–15 字符最佳）

## 🔍 五、tools 和 debugging 重叠时如何选择？

这是你最关心的问题，我给你一个永远不会错的判断法：

如果文章的核心是“解决问题” → debugging

例如：

- 如何通过堆栈定位崩溃
- 如何分析符号表找出错误位置

如果文章的核心是“工具使用说明” → tools

例如：

- addr2line 命令详解
- readelf 参数说明

如果两者都有：

- 主目录放 debugging
- tags 加上工具名

例如：

```
category: debugging
tags: [addr2line, symbol, elf, tools]
```

## 📘 六、知识库文件命名规范（推荐）

采用“英文技术词 + 中文描述”：

```
addr2line_符号表解析.md
backtrace_打印堆栈.md
perf_CPU分析.md
atomic_内存序.md
mutex_死锁分析.md
gdb_调试技巧.md
```

优点：

- 技术词英文 → 搜索方便
- 描述中文 → 一眼知道内容
